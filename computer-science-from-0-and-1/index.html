Computer Science from 0 (and 1)

06.10.2020 — Mathematics, Programming — 8 min read

This is meant to be a short introduction to some core topics and ideas in computer science. This is written with a mathematics-inclined reader in mind but does not assume any specific previous knowledge.

Building blocks

“A system of logical instructions that an automaton can carry out and which causes the automaton to perform some organized task is called a code.” — John von Neumann, The Computer and the Brain

As usual, in mathematics, we start with a set of primitives and some rules concerning them. There are multiple equivalent ways to begin this formalization, like lambda calculus or turing machines, however here I will another equivalent approach as it is easier to do the constructions in.

We will be working with a real world language, javascript (js), which has the added benefit of you being able to modify the code directly in your browser, you can modify any code in the live editors below! The whole expressive power of js is not necessary and we will be working with a really restricted subset of it, so the following explanation is general enough to work in almost every other programming language.

The following objects will be taken as a primitive.

  • Two truth values called true and false, also called bits.
  • The empty list [].
  • The natural numbers N\NN.
  • The bottom ⊥\bot⊥, which represents some sort of failure.
  • Infinite variables x1,x2,x3,...x_1, x_2, x_3,...x1​,x2​,x3​,.... For simplicity I will use other, more descriptive, names.

Furthermore, we have the following functions:

  • Two functions for the truth values, the binary function &&, called And, as well as the unary function !, Not. The tables below represent the outputs given some inputs. It should be clear that && is just the mathematicians ∧\wedge∧ and ! is just ¬\lnot¬.

Here are some comments on the construction so far. The only use of the natural numbers we have is to access the positions in the lists, we may as well have used letters or any other set provided that it is an infinite chain (so it is order-isomorphic to the naturals). Why bother introducing the push and accessor function if we won't ever use it? To emphasize that there is no weird magic going on, just application of some functions. In fact, nor push nor accessor explicitely exist in this way, but as we will see can be used with another syntax. As promised, you can interpret the bottom element ⊥\bot⊥ as representing a failure, in the case of the accessor function. Also, though we have defined our functions with some restricted domain (e.g. && only takes truth values), you could say that every object is in the domain of every function, just that if it is not in the restricted specified domain then the result is just ⊥\bot⊥.

We will introduce some more functions later, but even if we don't construct them explicitly it should be clear that all of them can be constructed out of these.

Creating functions and initializing variables

Javascript has multiple ways to specify functions, here we will take the lambda calculus approach: to define a function we write the arguments over which it will act, for example (x, y), add an arrow => after them, and finally write down the definition after it, called the body of the function. For example, here is how the disjunction (∨\vee∨) function can be written: (x, y) => !(!x && !y). As with any other function, you can apply it to some comma-separated arguments by appending them to the right, inside some parenthesis: ((x, y) => !(!x && !y))(true, false) would result in true.

The natural numbers

Finally, we are ready to begin the basic construction of natural numbers. The most obvious way to represent them in our restricted language is using binary numbers, e.g. 1102=22+21=6110_2=2^2+2^1=61102​=22+21=6. Now we can just take true to be equivalent to 111 and false ≅0\cong 0≅0. So a natural construction would be 6≅6\cong6≅[true, true, false], where clearly, a list of size nnn can only represent natural numbers up to 2n−12^n-12n−1. Importantly, we can do basic arithmetic with these natural numbers!

The halfAdd is a function that takes two bits and outputs their sum, expressed as a two-bits integer, it is called a half adder. This function and the others in this section are normally implemented directly into the circuits, using logic gates, because of how important and basic they are for almost every computation. As you would expect, there is a counterpart to the half adders, called full adders and they sum three values instead (the third one being a "carry").

This works, but it is certainly a mouthful and it's hard to understand, even if you know what a full adder is! To alleviate this problem, we will introduce some additional syntax. If the body of the function is wrapped in brackets, then the output of the function can be postponed until it is written after the return keyword. You sum the right most digits, and if there is a carry, you include in the second-to-last of the right digits, and so on. It is easy to generalize this to arbitrary n-bit integers, however notice that the output should be an (n+1)-bit integer because the sum could easily exceed the capacity of n-bit integers, just like in the threePlusTen example. This approach rapidly becomes unreasonable as every time you sum some numbers the memory required for storing them is doubled, so what most programming languages do is assume you won't ever exceed the maximum value, and output the sum of two n-bit numbers as another n-bit number. However if you actually try to sum above the maximum, this will cause an error and the program will fail. This type of errors are called overflows, and have cause quite amount of bugs, including failing the launch of the ´´Ariana 5 rocket which ended costing around $370 millions. For the rest of this article we will assume that the maximum value is enough and will sum and multiply without care.

Data types

Once understood the trick of the addition function, it's not hard to develop multiplication for 1-bit integers, then to 2-bit integers and generalize to n-bit integers. How about including negative integers? An easy approach is to reserve one location on the list, say the first, so that we interpret the number to be negative if that value is true and positive otherwise. This construction gives us signed n-bit integers, for example the signed 32-bit integers can take values from −231-2^{31}−231 to 231−12^{31}-1231−1. Similarly, it is possible to interpret characters (letters and symbols) as an n-bit integer, for example, see below how the USA telegraph code used to interpret 7-bit integers. Strings can be interpreted as a list of characters, and so on one could create constructions for rational numbers and floats (numbers with decimals).

Notice that all there is just true, false, and lists of these. The abstractions we are creating are called data types, and they form the basis for more and more complex programs. Of course, almost all real world programming language come with some already implemented data types and functions between them and javascript is no exception.
